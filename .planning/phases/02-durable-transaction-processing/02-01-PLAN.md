<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ingest route -> TransactionRepository | Untrusted transaction identifiers influence idempotency key construction and processing path |
| repository -> Mongo persistence | Processing state transitions cross durable storage boundary and can be raced by concurrent ingest attempts |
| repository -> event bus | Side-effect emission boundary must not be triggered for duplicate/replayed attempts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | Idempotency key builder | mitigate | Canonicalize and validate source/external IDs before key derivation; reject missing key components with explicit error |
| T-02-02 | R | Ledger transition history | mitigate | Persist deterministic timestamps and terminal status fields for each transition to support replay forensics |
| T-02-03 | D | Ledger claim flow | mitigate | Use unique idempotency index plus atomic claim filter (`RECEIVED/FAILED` only) to prevent concurrent duplicate processing |
| T-02-04 | E | Duplicate replay path | mitigate | Emit downstream event only when claim result indicates first processor; duplicates return stored payload without emit |
</threat_model>
