| Boundary | Description |
|----------|-------------|
| ingestion/detection payload -> RiskScorer | Untrusted detection context can influence segment and pattern dimensions used for tiering |
| threshold configuration -> policy evaluation | Misconfigured segment/pattern cutoffs can silently distort precision behavior |
| scorer output -> persisted Alert document | Incorrect confidence/tier mapping could mislead analyst triage decisions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | T | SegmentAwareThresholdPolicy input resolution | mitigate | Validate supported segment/pattern/geo-band keys and enforce bounded fallback defaults for unknown keys |
| T-03-02 | I | RiskScorer confidence derivation | mitigate | Keep confidence deterministic from score/evidence only and forbid random or external model calls in derivation path |
| T-03-03 | D | Policy cutoff expansion | mitigate | Bound numeric threshold ranges and reject invalid config values before classification to avoid pathological tier inflation |
| T-03-04 | R | Analyst-facing precision outcomes | mitigate | Persist confidence and policy context in alert records so results are reproducible and auditable by config lineage |
</threat_model>
