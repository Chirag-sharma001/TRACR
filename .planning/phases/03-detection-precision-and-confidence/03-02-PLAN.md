<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin client -> comparison endpoint | Untrusted query selectors and windows could force expensive aggregation paths |
| comparison service -> alert lineage data | Aggregation over lineage and confidence dimensions risks data disclosure if RBAC is bypassed |
| dashboard consumers -> API contract | Schema drift can break governance dashboards and hide precision regressions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | D | Comparison query params | mitigate | Clamp window bounds and validate selector params before building aggregation pipelines |
| T-03-06 | I | Admin comparison endpoint | mitigate | Reuse existing ADMIN middleware chain and expose only aggregate summaries (no raw alert payloads) |
| T-03-07 | T | Comparison schema contract | mitigate | Contract tests enforce stable before/after/delta keys with deterministic null-safe lineage handling |
| T-03-08 | R | Governance change decisions | mitigate | Include config_version_id and published_change_id linkage in comparison output to preserve audit traceability |
</threat_model>
